Problem 1 (5 pts) - Textbook Problem 4.7
================================================================================

The hint says the tent function can be generated by convolving two equal box
(rect) functions. So let me use that to find its Fourier transform.

Let rect(x) be a box function of width W:
    rect(x) = 1  for |x| <= W/2
            = 0  otherwise

Then the tent function t(x) is:
    t(x) = rect(x) * rect(x)    [* means convolution]

By the Convolution Theorem, convolution in the spatial domain becomes
multiplication in the frequency domain:
    FT{ f * g } = F(u) . G(u)

So:
    FT{ t(x) } = FT{ rect(x) * rect(x) }
               = FT{ rect(x) } . FT{ rect(x) }

The Fourier transform of a box function of width W is a sinc function:
    FT{ rect(x) } = W . sinc(Wu)    where sinc(x) = sin(pi*x) / (pi*x)

Therefore:
    FT{ t(x) } = W.sinc(Wu) . W.sinc(Wu)
               = W^2 . sinc^2(Wu)

This is a squared sinc, which matches the figure on the right. So the
Fourier transform of a tent function is indeed sinc^2. QED.

Problem 2 (5 pts)
================================================================================

The answer is NO - the histograms of the blurred images will NOT be equal,
even though the original histograms were the same.

Here is why:

The 3x3 averaging mask computes the average of each pixel's local neighborhood.
Even though both images have the same count of black (0) and white (255) pixels,
the spatial arrangement is completely different, and that is what determines how
the blur affects each image.

For Image 1 (left half black, right half white):
- Interior pixels on the black side: all 9 neighbors are 0, so average = 0
- Interior pixels on the white side: all 9 neighbors are 255, so average = 255
- Only the thin strip of pixels along the single vertical boundary gets
  intermediate values. This is only about 3 columns wide (~240 pixels out of 6400).
- So after blurring, most pixels are still 0 or 255.

For Image 2 (checkerboard pattern):
- In a 1x1 pixel checkerboard, if a pixel is black (0), its color rule is:
  pixel (i,j) is black when (i+j) is even, white when (i+j) is odd.
- For an interior black pixel at center of a 3x3 window:
    4-connected neighbors: (i+j)+-1 = odd -> all WHITE (255), 4 pixels
    diagonal neighbors: (i+j)+-2 or 0 = even -> all BLACK (0), 4 pixels
    average = (4 x 255 + 4 x 0 + 0) / 9 = 1020/9 ≈ 113
- For an interior white pixel at center:
    4-connected neighbors: all BLACK (0), 4 pixels
    diagonal neighbors: all WHITE (255), 4 pixels
    average = (255 + 4x0 + 4x255) / 9 = 1275/9 ≈ 142
- So almost every pixel in Image 2 becomes either ~113 or ~142 after blurring.

Comparing blurred histograms:
- Image 1: mostly 0 and 255, with a small number of intermediate values
- Image 2: mostly ~113 and ~142, with almost no 0 or 255 values

These are completely different histograms. So NO, they are not equal.

Problem 3 (5 pts)
================================================================================

The image segment is (row 0 = top, col 0 = left):

    Col:    0    1    2    3
    Row 0:  3    1    2    1   <- q is at (0,3), value = 1
    Row 1:  2    2    3    2
    Row 2:  1    2    1    1
    Row 3:  1    0    1    2   <- p is at (3,0), value = 1

------------------------------------------------------------
(a) V = {0, 1}
------------------------------------------------------------

First I need to identify which pixels have values in V = {0, 1}:
    (0,1)=1  (0,3)=1=q
    (2,0)=1  (2,2)=1  (2,3)=1
    (3,0)=1=p  (3,1)=0  (3,2)=1

Now check the neighbors of q = (0,3):
    4-neighbors: (0,2)=2 [not in V],  (1,3)=2 [not in V]
                 (row -1 and col 4 are out of bounds)
    8-neighbors add: (1,2)=3 [not in V]

q has NO neighbors in V under either 4 or 8 connectivity.
This means q is completely isolated - no path can ever reach it.

4-path from p to q: does NOT exist (q is isolated)
8-path from p to q: does NOT exist (q is isolated)

------------------------------------------------------------
(b) V = {1, 2}
------------------------------------------------------------

Pixels with values in V = {1, 2}:
    (0,1)=1  (0,2)=2  (0,3)=1=q
    (1,0)=2  (1,1)=2            (1,3)=2
    (2,0)=1  (2,1)=2  (2,2)=1  (2,3)=1
    (3,0)=1=p         (3,2)=1  (3,3)=2

Note: (3,1)=0 and (1,2)=3 are NOT in V - these act as obstacles.

Shortest 4-path:
The Manhattan distance from p=(3,0) to q=(0,3) is |3-0|+|0-3| = 6.
So we need at least 6 steps. Let me find a valid path of length 6:

    p=(3,0) -> (2,0) -> (2,1) -> (2,2) -> (2,3) -> (1,3) -> q=(0,3)
    Values:  1       1       2       1       1       2        1
    All in V and all 4-adjacent. Length = 6.

Since 6 = Manhattan lower bound, this is the shortest possible.
Shortest 4-path length = 6

Shortest 8-path:
The Chebyshev distance from p=(3,0) to q=(0,3) is max(3,3) = 3.
Can we do it in 3 steps? The only possible 3-step 8-path covering (-3,+3)
displacement is all diagonal NE steps:
    (3,0) -> (2,1) -> (1,2) -> (0,3)
But (1,2) = 3, which is NOT in V. So 3 steps is impossible.

Try 4 steps:
    p=(3,0) -> (2,1) -> (1,1) -> (0,2) -> q=(0,3)
    Values:  1       2       2       2        1
    Steps: diagonal, vertical, diagonal, horizontal. All valid.
    All values in V. Length = 4.

Shortest 8-path length = 4

Problem 5 (5 pts) - Textbook Problem 3.27
================================================================================

Unsharp masking formula:
    g(x,y) = f(x,y) + k * [f(x,y) - f_bar(x,y)]

Using k = 1:
    g = f + (f - f_bar) = 2f - f_bar

The given averaging filter is (1/9) times the 3x3 all-ones mask, so:
    f_bar(x,y) = (1/9) * [sum of all 9 pixels in 3x3 neighborhood]

Expanding:
    g(x,y) = 2*f(0,0) - (1/9)*[f(-1,-1) + f(-1,0) + f(-1,1)
                                + f(0,-1)  + f(0,0)  + f(0,1)
                                + f(1,-1)  + f(1,0)  + f(1,1)]

Separating the center term:
    g(x,y) = 2*f(0,0) - (1/9)*f(0,0) - (1/9)*[8 surrounding neighbors]
           = (18/9 - 1/9)*f(0,0) - (1/9)*[8 surrounding neighbors]
           = (17/9)*f(0,0) - (1/9)*[8 surrounding neighbors]

Writing this as a single 3x3 mask:

         1  | -1  -1  -1 |
    h = ---  | -1  17  -1 |
         9  | -1  -1  -1 |

Verification: mask sum = (1/9)(17 + 8*(-1)) = (1/9)(9) = 1
A sum of 1 means the mean intensity is preserved, which is correct.

Problem 6 (5 pts) - Textbook Problem 3.28
================================================================================

Given Laplacian definition:
    nabla^2 f(x,y) = f(x+1,y) + f(x-1,y) + f(x,y+1) + f(x,y-1) - 4f(x,y)

I need to show that f - nabla^2(f) is proportional to unsharp masking.

Starting with f - nabla^2(f):
    f - nabla^2 f
    = f(x,y) - [f(x+1,y) + f(x-1,y) + f(x,y+1) + f(x,y-1) - 4f(x,y)]
    = f(x,y) - f(x+1,y) - f(x-1,y) - f(x,y+1) - f(x,y-1) + 4f(x,y)
    = 5f(x,y) - [f(x+1,y) + f(x-1,y) + f(x,y+1) + f(x,y-1)]

Now define f_bar as the average of the 4 direct neighbors:
    f_bar(x,y) = (1/4) * [f(x+1,y) + f(x-1,y) + f(x,y+1) + f(x,y-1)]

So the sum of 4 neighbors = 4 * f_bar.

Substituting:
    f - nabla^2 f = 5f(x,y) - 4*f_bar(x,y)
                  = f(x,y) + 4f(x,y) - 4*f_bar(x,y)
                  = f(x,y) + 4*[f(x,y) - f_bar(x,y)]

The unsharp masking formula is:
    g = f + k*(f - f_bar)

Our result matches this with k = 4 and f_bar = average of 4 neighbors.

Therefore subtracting the Laplacian from an image is equivalent to
unsharp masking with strength k = 4. QED.

Note: the f_bar here uses only the 4 direct neighbors (not a full 3x3 box),
because the given Laplacian only involves 4 neighbors.

Problem 7 (5 pts) - Textbook Problem 4.12
================================================================================

The checkerboard has 1mm x 1mm squares. As the hint says, viewing along any
row (or column) the pattern looks like:

    ... 1  0  1  0  1  0  ...

where each value lasts exactly 1mm. This is a periodic signal with:
    Period  T = 2mm   (one black + one white = one full cycle)
    Frequency  f = 1/T = 1/2mm = 0.5 cycles/mm

By the Nyquist-Shannon Sampling Theorem, to avoid aliasing we need:
    sampling rate  f_s >= 2 * f_max
                       = 2 * 0.5
                       = 1 sample/mm

Minimum sampling rate = 1 sample/mm (applied in both x and y directions)

Physical intuition: at 1 sample/mm we place exactly one sample inside each
1mm x 1mm square, which is just enough to distinguish every black and white
square without aliasing.

Problem 8 (5 pts) - Textbook Problem 4.27
================================================================================

The 3x3 spatial mask that averages only the 4 closest neighbors (excluding
the center point) is:

         0    1/4   0
        1/4    0   1/4
         0    1/4   0

------------------------------------------------------------
(a) Finding H(u,v)
------------------------------------------------------------

Writing h(x,y) using shifted impulses (centered at origin):
    h(x,y) = (1/4)*d(x-1,y) + (1/4)*d(x+1,y) + (1/4)*d(x,y-1) + (1/4)*d(x,y+1)

where d(x,y) is the unit impulse (1 at origin, 0 elsewhere).

Taking the 2D DFT (the DFT of a shifted impulse d(x-a,y-b) is e^{-j2pi(ua+vb)/N}):

    H(u,v) = (1/4)*e^{-j2pi*u/N}  +  (1/4)*e^{+j2pi*u/N}
           + (1/4)*e^{-j2pi*v/N}  +  (1/4)*e^{+j2pi*v/N}

    H(u,v) = (1/4) * [e^{-j2pi*u/N} + e^{+j2pi*u/N}] + (1/4) * [e^{-j2pi*v/N} + e^{+j2pi*v/N}]

Using Euler's formula  e^{jx} + e^{-jx} = 2cos(x):

    H(u,v) = (1/4)*2*cos(2pi*u/N) + (1/4)*2*cos(2pi*v/N)

    H(u,v) = (1/2) * [cos(2*pi*u/N) + cos(2*pi*v/N)]

------------------------------------------------------------
(b) Showing H(u,v) is a low-pass filter
------------------------------------------------------------

Evaluate H along the u-axis by setting v = 0:
    H(u, 0) = (1/2) * [cos(2*pi*u/N) + 1]

Using the half-angle identity  (1 + cos(theta))/2 = cos^2(theta/2):
    H(u, 0) = cos^2(pi*u/N)

Checking values from DC to highest frequency:
    u = 0   :  H = cos^2(0)     = 1.000   <- maximum (DC passes fully)
    u = N/8 :  H = cos^2(pi/8)  = 0.854
    u = N/4 :  H = cos^2(pi/4)  = 0.500
    u = 3N/8:  H = cos^2(3pi/8) = 0.146
    u = N/2 :  H = cos^2(pi/2)  = 0.000   <- highest frequency is blocked

H(u,0) decreases monotonically from 1 to 0 as u increases from 0 to N/2.
The same applies along the v-axis by symmetry.

This shows the filter passes low frequencies (near DC) and attenuates high
frequencies (near Nyquist) - which is exactly the definition of a low-pass
filter.

Note: at the corner frequency (u=N/2, v=N/2):
    H(N/2, N/2) = (1/2)[cos(pi)+cos(pi)] = (1/2)(-1-1) = -1
This means the pure diagonal checkerboard frequency is inverted (not passed
normally). However along the principal horizontal and vertical frequency axes,
the filter is clearly low-pass, and in practice this filter smooths/blurs
images, confirming low-pass behavior.
